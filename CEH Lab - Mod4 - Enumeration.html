Lab 1: Perform NetBIOS Enumeration using nbtstat

NetBIOS enumeration is a process of obtaining sensitive information about the target such as a list of computers belonging to a target domain, network shares, policies, etc.

First step in the enumeration of a Windows system is to exploit the NetBIOS API

Windows uses NetBIOS for file and printer sharing

NetBIOS name: Unique computer name assigned to Windows systems - 16-character ASCII string that identifies the network device over TCP/IP. The first 15 characters are used for the device name, and the 16th is reserved for the service or name record type.
Runs on Windows systems even when not in use.

NetBIOS enumeration allows attackers to read or write to a remote computer system (depending on the availability of shares) or launch a denial of service (DoS) attack.

Lab Objectives:

Perform NetBIOS enumeration using Windows command-line utilities

Nbtstat helps in troubleshooting NETBIOS name resolution problems.

nbtstat -a <Target IP>

-a displays the NetBIOS name table of a remote computer

nbtstat -c

-c displays name cache of remote computer

net use

Displays info such as connection status, shared folder/drive and network info

Lab 2: Perform SNMP Enumeration using snmpWalk

SNMP (Simple Network Management Protocol) is an application layer protocol that runs on UDP (User Datagram Protocol) and maintains and manages routers, hubs, and switches on an IP network.

SNMP agent and SNMP Management station

We use SNMP to create a list of user accounts and devices on target computer

Can extract information about:

Network resources (such as hosts, routers, devices, and shares)

Network information (such as ARP tables, routing tables, device-specific information, and traffic statistics

Lab Objectives:

Perform SNMP enumeration using snmpWalk

snmpWalk is a command line tool that scans SNMP nodes and identifies variables available for accessing the target network

snmpwalk -v1 -c public [target IP]

-v: Version

-c: sets Community string

Displays OIDs, variables, and other info

snmpwalk -v2c -c public [Target IP Address]

Displays data transmitted from the SNMP agent to the SNMP server, including information on server, user credentials, and other parameters

Lab 3: Perform LDAP Enumeration

Uses LDAP to generate a list of distributed directory services on the target system.

LDAP (Lightweight Directory Access Protocol) is an Internet protocol for accessing distributed directory services over a network

Uses DNS for lookups and resolution of queries

Client starts LDAP session by connecting to DSA on TCP port 389

BER (Basic Encoding Rules) used to transmit info between client and server

Can anonymously query the LDAP service for sensitive information:

Usernames, addresses, departmental details, and server names

Lab Objectives:

Perform LDAP enumeration using Active Directory Explorer (AD Explorer)

  1. Open AD Explorer
  1. Enter Target IP Adress
  1. Expand DC=C:EH, DC=com > Expand CN=Users
  1. Click on a User > Right click any property in the right pane > Modify > Can change the value

Other LDAP Enumeration Tools:

Lab 4: Perform NFS Enumeration using RPCscan and SuperEnum

NFS enumeration is a method by which exported directories and shared data on target systems is extracted.

NFS (Network File System) is a type of file system that enables computer users to access, view, store, and update files over a remote server.

RPCScan communicates with RPC (remote procedure call) services and checks misconfigurations on NFS shares.

SuperEnum includes a script that performs a basic enumeration of any open port

To determine if NFS port is open and NFS service is up:

nmap -p 2049 <target IP>

SuperEnum

Note: If error running ./superenum script - execute chmod +x superenum

  1. View open ports and services

RPCscan:

  1. cd RPCScan
  1. python3 rpc-scan.py <Target IP Addresses> --rpc

Lab 5: Perform DNS Enumeration using Zone Transfer

DNS enumeration is a process that locates and lists all possible DNS records for a target domain, including usernames

Yields information such as:

DNS server names

Hostnames

Machine names

Usernames

IP addresses

Aliases

Zone transfer: Process of transferring a copy of the DNS zone file from the primary DNS server to a secondary DNS server

In most cases, the DNS server maintains a spare or secondary server for redundancy, which holds all information stored in the main server.

If DNS transfer setting enabled on target DNS server - will give DNS information

If not - will return an error saying it has failed or refuses the zone transfer

Linux / dig:

  1. dig ns <Target Domain>
    1. Retrieves info about all the DNS name servers of the target domain and displays it in the ANSWER SECTION
    1. Ex. ns2.bluehost.com is returned
  1. dig @<NameServer> <Target Domain> axfr
    1. axfr retrieves zone information
    1. This will show if Transfer failed/refused or if enabled

Windows / nslookup:

  1. nslookup
  1. set querytype=soa
    1. This retrieves administrative info about DNS zone of target domain
  1. <Target Domain>
  1. ls -d <Name Server>
    1. Requests a zone transfer of specified name server

Lab 6: Perform SMTP Enumeration using Nmap

SMTP enumeration determines a valid list of user accounts on the SMTP server.

Obtain:

Valid users

Delivery addresses

Message recipients


Simple Mail Transfer Protocol (SMTP) is an internet standard based communication protocol for electronic mail transmission

  • Uses mail exchange (MX) servers to direct mail via DNS
  • Runs on TCP port 25, 2525, or 587

Lab Objectives:

Perform SMTP enumeration using Nmap

  1. nmap -p 25 --script=smtp-enum-users <Target IP>
    1. Uses NSE to display list of all possible mail users on target machine
  1. nmap -p 25 --script=smtp-open-relay <Target IP>
    1. Uses NSE to display list of open SMTP relays on target machine
  1. nmap -p 25 --script=smtp-commands <Target IP>
    1. Displays list of all SMTP commands available

Goal:

Attackers can perform password spraying attacks to gain unauthorized access to user accounts
